# claude-rules-refactor

> 规范来源：https://linux.do/t/topic/1871216

帮你维护 Claude Code 的 CLAUDE.md 配置体系，支持四种模式。

## 功能

### 模式 A — 重构
把臃肿的 CLAUDE.md 按规范拆成模块化规则文件：
- 识别模糊指令并具体化（"代码要简洁" → "函数不超过 30 行"）
- 按主题拆分到独立规则文件，每文件 ≤ 50 行
- CLAUDE.md 只保留全局偏好 + `@` 引用列表

### 模式 B — 从会话挖掘
扫描 `~/.claude/projects/**/*.jsonl` 历史 session，提炼值得写进配置的规则：
- 提取用户纠正、工具报错、反复提醒等信号
- 归纳候选规则并附上原始证据
- 用户确认后写入对应规则文件

### 模式 C — 健康检查
审计现有规则文件，输出带评分的诊断报告：
- 模糊指令、文件超长、主题混杂
- 重复规则、全局/项目错位
- 孤立引用（`@` 了不存在的文件）
- 跨项目重复（应提升为全局规则）

### 模式 D — 建立项目级 CLAUDE.md
从零初始化一个项目的 Claude 配置：
- 探索项目技术栈（语言、框架、测试工具、构建工具）
- 推断项目特定规则，不重复全局配置
- 生成 `.claude/CLAUDE.md` + `rules/stack.md` + `rules/workflow.md`
- 写入前预览，确认后执行

## 推荐工作流

```
C（先诊断现状）→ B（挖掘新规则）→ A（统一重构）
```

## 触发词

`优化claude.md` / `重构claude.md` / `更新全局配置` / `整理规则文件` /
`从会话数据提炼规则` / `规则健康检查` / `规则冲突` / `跨项目同步规则` /
`给项目建claude.md` / `初始化项目配置`
